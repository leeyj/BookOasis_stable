# BookOasis Handover & Development Roadmap

이 문서는 BookOasis 미디어 서버의 1단계 리팩토링 및 2단계 독립 마이그레이션이 완료된 이후, 후속 개발 단계의 스케일링, 안정성, 모바일 뷰어 고도화 및 최적화 로드맵을 정의한 명세입니다.

---

## [Phase 3] VFS / 모바일 스케일 및 뷰어 최적화

### 1. VFS 네트워크 레이턴시 극복을 위한 비동기 VFS 프리페치 및 스캔 최적화
- **기능 개요**: rclone 구글 드라이브 마운트 경로(VFS)는 파일 메타데이터 및 헤더 탐색 시 네트워크 왕복 레이턴시가 수십 초 이상 걸려 전체 스캐너 및 리포트 생성을 지연시키고 Worker Timeout을 발생시킵니다. 이를 비동기 캐싱 및 백그라운드 프리페치(Prefetch) 파이프라인으로 구조화합니다.
- **구현 제안**:
  - **백그라운드 VFS 디렉토리 캐시 데몬**: 시스템 시동 시 VFS 경로 구조를 메모리에 캐싱하고, OS 파일 탐색 이벤트(inotify 등)를 받거나 1시간 주기로 변경사항만 긁어 백그라운드 캐시를 동기화합니다. 스캐너는 로컬 캐시를 즉각 조회하여 실시간 I/O 대기시간을 0초에 수렴시킵니다.
  - **비동기 압축파일 헤더 프리페치**: 책 로딩/스캔 시 해당 책의 오프셋 분석(ZIP 중앙 디렉토리 조회) 및 PDF/EPUB 메타 파싱을 별도 스레드로 백그라운드에서 백오프(Backoff) 알고리즘을 태워 큐 형태로 실행합니다.

### 2. 모바일 웹 뷰어 성능 튜닝 및 메모리 릭(Leak) 누수 차단
- **기능 개요**: 고해상도 만화 ZIP 파일이나 PDF 도서를 볼 때 모바일 사파리(iOS) 및 크롬(Android) 브라우저는 가용 RAM 한계가 엄격하여 DOM 메모리 부족으로 탭이 크래시되거나 리로드되는 장애를 방지합니다.
- **구현 제안**:
  - **이미지 뷰어 가상 스크롤(Virtual Scrolling) 및 DOM 재활용**: 뷰어 내 로드된 만화/PDF 페이지 이미지 DOM을 무제한 누적하지 않고, 뷰포트 기준 상하 2~3페이지만 실제 DOM을 활성화하며 이전 이미지는 HTML DOM에서 파괴하여 RAM 점유율을 50MB 이내로 고정시킵니다.
  - **Pillow 인코딩 단계 WebP 해상도 리사이징 다운그레이드**: 모바일 기기가 접속한 경우, `/api/media/view` 요청 시 WebP 가로폭 해상도를 모바일 최적 규격(1080px 내외)으로 온디맨드 다운사이징 렌더링을 태워 트래픽 및 기기 디코딩 연산 부담을 최소화합니다.

### 3. ZIP / PDF 대량 도서 스캔을 위한 백그라운드 [Lazy 표지 스캔] 도입
- **기능 개요**: 다량의 고해상도 PDF 및 원격지 ZIP 도서를 스캔하는 도중 표지를 즉시 일괄 파싱하면 엄청난 양의 I/O 병목 및 메모리 누적으로 gunicorn Worker가 사살(SIGKILL)되는 OOM 장애가 일어납니다. 이를 우회하기 위해 **4. 표지 스캔(Lazy)** 파이프라인을 추가합니다.
- **구현 제안**:
  - **스캔 중 즉시 추출 생략**: 라이브러리 대량 스캔 중 원격지 VFS 경로이거나 PDF 포맷 도서인 경우 표지 이미지 빌드를 완전히 생략하고 `cover_image = None`으로 DB를 우선 기록합니다.
  - **LazyCoverWorker 백그라운드 데몬 기동**: 미디어 서버 시작 시 백그라운드에서 1개 스레드로 동작하는 데몬 워커를 구동합니다.
  - **1회 1책 Throttle & Release 처리**: DB에서 표지가 빈 도서를 1권씩만 조회하여 표지 WebP를 추출하고, 추출 즉시 `fitz` 및 `Pillow` 사용 변수 `del` 처리 및 `gc.collect()` 호출을 수행합니다. 다음 도서 처리까지 `time.sleep(3.0)` 간격을 두어 CPU 및 RAM의 급격한 점유 스파이크를 원천적으로 방지합니다.

## [Phase 4] 관리 기능 고도화 및 OPDS 표준 연동

### 4. 마이그레이터(migrator.py) 고속화 튜닝
- SQLite DB I/O 병목 완화를 위한 `PRAGMA synchronous = OFF;`, `PRAGMA journal_mode = MEMORY;` 적용.
- 트랜잭션 단위 대형화 및 `executemany`를 통한 벌크 인서트 구현.

### 5. 메타 정보 편집 및 수동 표지 교체
- 상세 뷰 내 메타데이터(제목, 작가, 출판사 등)를 편집할 수 있는 프론트엔드 UI/백엔드 API 구축.
- 이미지 파일 업로드 시 `book_{hash}.webp`를 강제 Overwrite 하고 브라우저 캐시 버스팅을 위한 `cover_updated_at` 쿼리 파라미터 적용.

### 6. 파일 관리자 로그인 세션 및 보안 설계
- Flask session 또는 JWT 기반의 관리자 로그인 화면 구성 및 보안 미들웨어 구현.
- LFI/Path Traversal 방지를 위한 `os.path.commonpath` 상위 경로 탈출 검증 로직 구현.

### 7. OPDS 규격 RESTful 엔드포인트 연동
- KyBook, YACReader, Kuro Reader 등 외부 모바일 리더 연동용 OPDS XML 피드(`/opds/v1/feed`) 제공.

### 8. 다국어(i18n) 설계
