# BookOasis vs Stump 비교자료 (최신화)

작성일: 2026-07-17  
범위: Stump 공식 문서(https://www.stumpapp.dev/docs)와 BookOasis 저장소 문서 기준의 사실 비교

## 1) 비교 원칙

- 마케팅 표현/주관 평가 대신 문서로 확인 가능한 사실만 기재
- 성능 수치(RAM, CPU, 속도)는 동일 벤치마크 미실시 상태이므로 단정하지 않음
- 불명확 항목은 `미확인(검증 필요)`로 표기

## 2) 솔루션 개요

| 항목 | Stump | BookOasis |
| :--- | :--- | :--- |
| 포지셔닝 | 오픈소스 셀프호스트 디지털 북/코믹 미디어 서버 | 오픈소스 셀프호스트 디지털 북/코믹 미디어 서버 |
| 기술 스택(문서/코드 기준) | Rust 서버 + React 계열 클라이언트 | Python(Flask) + Vanilla JS |
| 라이선스 | 오픈소스(공식 저장소 기준) | AGPL v3 (README 배지/문서 기준) |

## 3) 기능 비교 (문서 확인 기반)

| 항목 | Stump | BookOasis |
| :--- | :--- | :--- |
| 설치 방식 | 바이너리, Docker, 소스 빌드 문서 제공 | Docker, Native Python 문서 제공 |
| 웹 앱 | 제공 (SPA) | 제공 |
| 모바일 앱 | iOS/Android 베타 앱 문서 제공, 오프라인 읽기/동기화 명시 | 전용 모바일 앱 문서 없음, 모바일 웹 + OPDS 연동 중심 |
| 데스크톱 앱 | 별도 데스크톱 앱(나이틀리) 문서 제공 | 별도 데스크톱 앱 문서 없음 |
| 지원 포맷(공식 표기) | EPUB, PDF, CBZ/ZIP, CBR/RAR 등 | ZIP/CBZ, EPUB, PDF, TXT(문서 기준) |
| OPDS | OPDS 1.2/2.0 명시, OPDS 관련 API 키 경로 제공 | OPDS 및 app-opds 엔드포인트 제공(문서/API 명세) |
| 인증 방식 | 세션 + JWT + API Key + OPDS Basic Auth(문서 명시) | Flask 세션 + OPDS Basic Auth(문서/코드 명시) |
| OIDC/SSO | OIDC 공식 가이드 및 설정 키 문서 제공 | Proxy Header Auth(리버스 프록시 기반) 가이드 제공 |
| 권한 모델 | 사용자/권한 enum 다수, 라이브러리 접근 제어 문서 제공 | Admin/User + 성인 접근/라이브러리 권한 관리 문서 제공 |
| API 형태 | GraphQL 중심 + 일부 REST | REST 중심 |
| CLI | 서버 실행파일 기반 CLI 문서 제공 | 별도 CLI 사용자 문서 미확인 |
| 플러그인 확장 | 통합/기능 확장 문서 다수 (KoReader/Kobo 등) | metadata/dashboard/context-menu 플러그인 가이드 제공 |
| 웹훅/이벤트 | 미확인(문서 내 확인 필요) | `book.new/read/finish` 이벤트 웹훅 문서화 |

## 4) 운영 관점 비교

| 항목 | Stump | BookOasis |
| :--- | :--- | :--- |
| 세션 만료/정리 | 세션 저장 및 만료 정리 인터벌 문서 명시 | 유휴시간 자동 로그아웃 기본 구현 미확인(현 구조상 별도 설정 필요) |
| OPDS 진행률 처리 | OPDS progression 관련 설정 키 문서 존재 | OPDS 진행률 API 및 포맷별 처리 문서 존재 |
| 업로드 기능 | 업로드 기능(옵트인) + 권한/보안 고려 문서 제공 | 업로드는 플러그인/관리 흐름 중심(일반 업로드 기능은 미확인) |
| 다중 인스턴스/클라이언트 | 모바일/데스크톱에서 멀티 서버 연결 문서 존재 | 멀티 서버 클라이언트 문서 미확인 |

## 5) 강점/주의점 (객관 요약)

### Stump
- 강점
	- 공식 문서 범위가 넓고(설치/권한/OIDC/API/CLI/앱), 인증 옵션이 다양함
	- 전용 모바일/데스크톱 앱 제공 흐름이 명확함
- 주의점
	- 일부 앱(모바일/데스크톱)은 베타/나이틀리 전제 안내가 존재

### BookOasis
- 강점
	- Python 기반 단순 배포 흐름, OPDS/app-opds 실사용 문서가 구체적
	- 메타데이터/대시보드/컨텍스트 메뉴 플러그인 계약이 명확함
	- 한국어 환경(예: 알라딘 연동) 중심 운영 문서가 상세함
- 주의점
	- 전용 모바일/데스크톱 앱 문서 범위는 상대적으로 제한적
	- 유휴시간 자동 세션아웃은 기본 구현으로 확인되지 않음

## 6) 결론 (도입 시나리오 기준)

- 앱 생태계(모바일/데스크톱), 인증 옵션(OIDC/JWT/API Key), GraphQL/CLI 중심 운영을 중시하면: Stump가 더 적합할 가능성이 높음
- Python/Flask 기반 단순 운영, OPDS 실사용, 한국어 메타데이터 플러그인 확장을 중시하면: BookOasis가 더 적합할 가능성이 높음

## 7) 검증 필요 항목 (공정 비교를 위해 권장)

- 동일 데이터셋(예: 1만권, 10만권) 기준 스캔 시간/메모리/CPU 벤치마크
- OPDS 클라이언트별(문리더, Mihon, KOReader) 진행률/호환성 회귀 테스트
- 라이브러리 권한 모델(기본 허용/기본 차단)의 운영 정책 적합성 검토
