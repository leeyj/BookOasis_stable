# 📊 미디어 서버 비교 분석: Kavita vs Komga vs BookOasis

이 문서는 대표적인 오픈소스 만화/도서 미디어 서버인 **Kavita**, **Komga**, 그리고 초경량 아키텍처로 개발된 **BookOasis**의 기능, 성능, 도서 관리 능력을 객관적으로 비교한 자료입니다.

## 1. 핵심 아키텍처 및 철학 비교

| 항목 | Kavita | Komga | BookOasis |
| :--- | :--- | :--- | :--- |
| **개발 언어** | C# (.NET) + Angular | Java (Spring Boot) + Vue.js | Python (Flask) + Vanilla JS |
| **의존성/무게** | 무거움 (방대한 프레임워크) | 매우 무거움 (JVM 환경 필수) | **초경량** (노빌드, 서드파티 최소화) |
| **스트리밍 방식** | 캐시 기반 압축 해제 및 전송 | 캐시 기반 전처리 및 전송 | **오프셋 기반 실시간 다이렉트 전송** |
| **설치 편의성** | 보통 (Docker 권장) | 보통 (Docker 권장) | **최상** (Native Python만으로도 1분 컷) |

---

## 2. 성능 (Performance) 비교

BookOasis는 극단적인 최적화를 목표로 설계되었기 때문에 리소스 사용량과 체감 속도에서 압도적인 우위를 가집니다.

| 항목 | Kavita | Komga | BookOasis |
| :--- | :--- | :--- | :--- |
| **디스크 I/O** | 높음 (스캔/읽기 시 압축 해제 잦음) | 매우 높음 (백그라운드 캐싱 빈번) | **최저** (물리적 압축 해제 100% 배제) |
| **RAM 점유율** | 높음 (500MB ~ 1GB 이상) | 매우 높음 (1GB ~ 2GB 이상) | **초저전력** (수십 MB 수준 유지) |
| **뷰어 로딩 속도**| 보통 (캐시가 없을 시 지연 발생) | 보통 (초기 로딩 시 딜레이 존재) | **즉각적 (Zero-Delay)** |
| **대용량 스캔** | 다소 느림 (메타데이터 심층 파싱) | 다소 느림 (정교한 분석 수행) | **매우 빠름** (필수 정보 및 오프셋만 고속 추출) |

> [!TIP]
> **성능 총평**: 라즈베리파이나 저사양 NAS 등 극한의 환경에서는 **BookOasis**가 압도적인 효율을 발휘합니다. Kavita와 Komga는 풍부한 기능을 위해 무거운 런타임과 높은 메모리를 요구합니다.

## 3. 기능 (Features) 비교

| 항목 | Kavita | Komga | BookOasis |
| :--- | :--- | :--- | :--- |
| **지원 포맷** | ZIP, CBZ, CBR, EPUB, PDF 등 | ZIP, CBZ, CBR, PDF, EPUB 등 | **ZIP, CBZ, PDF, TXT** |
| **서드파티 연동** | OPDS, 다양한 외부 리더기 완벽 지원 | OPDS, Tachiyomi 등 생태계 방대 | OPDS 지원 (Tachiyomi 등 연동 가능) |
| **UI/UX** | 매우 화려하고 현대적 | 깔끔하고 정돈된 디자인 | 미니멀, 모바일 최적화 (Vanilla JS) |
| **플러그인 확장** | 자체 내장 로직 의존 | 자체 내장 로직 의존 | **웹 UI 기반 동적 플러그인 아키텍처** |
| **성인/권한 제어**| 지원 (계정별 라이브러리 할당) | 지원 (세밀한 권한 제어 가능) | 지원 (계정별 성인 인증 플래그 격리) |

> [!NOTE]
> **기능 총평**: EPUB, CBR 등 다양한 포맷 지원과 범용성 측면에서는 **Kavita**와 **Komga**가 우세합니다. BookOasis는 범용성보다는 **만화(ZIP/CBZ)와 텍스트(TXT)** 스트리밍에 집중하여 선택과 집중을 하였습니다.

## 4. 도서 관리 능력 (Book Management) 비교

| 항목 | Kavita | Komga | BookOasis |
| :--- | :--- | :--- | :--- |
| **자동 메타데이터**| ComicInfo.xml, EPUB 메타 자동 파싱 | ComicInfo.xml 파싱 매우 강력함 | **ComicInfo.xml 기본 파싱 지원** + 외부 API(알라딘 등) 플러그인 연동 |
| **시리즈 그룹화** | **최상** (이름 규칙 및 메타로 완벽히 묶음) | **최상** (컬렉션, 시리즈 관리 탁월) | 보통 (디렉터리/폴더 구조에 주로 의존) |
| **에러 파일 처리** | 스캔 중단되거나 묵시적 무시 | 에러 로그 남김 | **전용 '에러 리포트' 아키텍처 아카이빙** |
| **한국어 친화성** | 다국어 지원하나 한국 메타 부족 | 다국어 지원하나 한국 메타 부족 | **최상 (국내 전용 알라딘 API 완벽 연동)** |

> [!IMPORTANT]
> **관리 능력 총평**: 
> * **Kavita / Komga**: 서양 만화(코믹스)의 `ComicInfo.xml` 기반으로 시리즈를 자동 그룹화하고 컬렉션을 만드는 **내부 메타데이터 관리 시스템이 매우 정교**합니다.
> * **BookOasis**: 복잡한 메타데이터 파싱 로직을 과감히 덜어내는 대신, **사용자의 물리적 폴더 구조를 그대로 반영**하며, 부족한 정보는 **알라딘 API 플러그인**을 통해 한국 실정에 맞는 메타데이터를 원클릭으로 주입하는 "실용성"에 초점을 맞췄습니다.

## 💡 결론 요약 (Conclusion)

1. **Komga**: 가장 안정적이고 거대한 생태계를 가진 **표준적인 만화 서버**. 서버 스펙(RAM)이 넉넉하고 서양 코믹스 중심의 `ComicInfo.xml` 관리를 원할 때 추천.
2. **Kavita**: EPUB 소설과 만화를 아우르는 **올라운더 미디어 서버**. 화려한 UI와 다중 포맷 지원이 필요할 때 추천.
3. **BookOasis**: 극한의 속도와 가벼움을 추구하는 **실전 지향형 초경량 서버**. 복잡한 설정 없이 내 하드디스크 폴더 그대로, 메모리 낭비 없이 스마트폰에서 "가장 빠르게" 만화를 스트리밍하고 싶을 때 (특히 한국 도서 메타 연동이 필요할 때) 추천.
